从零搭建 Github Pages

Aug 09, 2020

创建仓库

首先需要在 Github 中创建一个名为 <username>.github.io 的仓库，其中 <username> 为你的 Github 用户名。

注意：
由于 username 不方便叙述，故下文中均以 sudrizzz 为例替代 username，
请根据实际情况进行更改。

例如我的 Github 用户名是 sudrizzz，于是创建的仓库名就是 sudrizzz.github.io。

初始化 Hugo

安装 Hugo 应用

https://github.com/gohugoio/hugo/releases

配置环境变量

将安装 Hugo 的目录路径配置到用户环境变量 PATH 中，如图

检验配置

在命令行窗口中输入以下内容

1hugo version

如果得到类似以下的结果则说明配置正确。

Hugo Static Site Generator v0.74.3-DA0437B4 windows/amd64 BuildDate: 2020-07-23T16:23:30Z

创建博客

在命令行中输入以下命令

hugo new site <blog_name>

上述命令将会创建一个名为 blog_name 的文件夹，请按照个人喜好取名。按照我的博客为例，取名为 sudrizzz.github.io，这样也方便后续进行代码提交

添加主题

以 yinyang 主题为例，按照文档中的安装步骤，执行以下命令

1cd sudrizzz.github.io
2git clone git@github.com:joway/hugo-theme-yinyang.git themes/yinyang

进入 sudrizzz.github.io/themes/yinyang/exampleSite 目录，将 content 文件夹和 config.toml 文件拷贝到 sudrizzz.github.io 目录。

自定义主题

根据 yinyang 主题配置文档，我们可以按需修改其中的内容即可，以下贴出我自定义的配置内容。

32[[params.socials]]
33name = "RSS"
34link = "/index.xml"

添加文章

在 sudrizzz.github.io 文件夹中执行以下命令以新增文章

1hugo new content/cn/posts/my-first-post.md

但是我个人不建议这么操作，原因有两点

  1. 此主题的 posts 目录分为 cn 和 en，也就是中文和英语目录，路径较长，敲命令不是很方便；
  2. md 文件中需要包含特定格式的头信息（如下），才能被正常渲染，而通过上述命令生成的 md 文件是空白文件，自己添加头信息也不太现实。
md 头信息

title: "从零搭建 Github Pages"
date: 2020-08-09T00:00:00+08:00
draft: false
categories: ["blog"]
slug: "build-blog-from-scratch"

其中各项含义如下
title: 文章显示标题
date: 文章创建日期与时间
draft: 是否为草稿状态
categories: 分类信息，可添多个关键词，用半角逗号分隔
slug: 文章渲染后的 url

所以我更推荐直接拷贝已有的文章，然后修改其中的内容（包括头信息）即可。 在此我创建了名为 TestFile.md 的测试文件以作演示，完整内容如下。

---
title: "这是一个测试文件"
date: 2020-01-01T00:00:00+08:00
draft: false
categories: ["test"]
slug: "this-is-a-test-file"
---

This is a test file.
Nothing special here.

编译文章

这样就编译完成了，编译后生成的静态文件放在 sudrizzz.github.io/public 目录下。结构如下：

├─categories
│  └─test
├─css
│  └─highlight
├─fonts
├─images
├─js
├─posts
│  └─this-is-a-test-file
└─tags

至此博客就基本搭建完成了。接下来要做的就是与前文中提到的 Github Pages 仓库结合起来。

推送到 Github

说明 1：
此处涉及到 Git 的相关操作，请先根据其他教程配置好 Git 环境再执行命令；
或者也可以安装一些 Git 可视化工具来简化操作，例如 TortoiseGit。

说明 2:
我们只需要将 Hugo 生成的静态文件托管到 Github，
所以请确保下述命令是在 /pulic 目录中执行的。

在 sudrizzz.github.io/pulic 目录中执行以下命令

1git clone https://github.com/sudrizzz/sudrizzz.github.io.git
2git add .
3git commit -m "<commit_info>"
4git push origin master

这样博客的所有内容就都已经推送到了 Github，稍等片刻，即可访问 sudrizzz.github.io 查看部署结果。

总结

前文搭建的博客是由 Github Pages + Hugo 实现的，具体的新增文章步骤可以总结为以下几点。

  1. 在 content/posts/cn/ 目录中新增文章，修改相关头信息，完成内容创作并保存；
  2. 执行 hugo server 并访问 localhost:1313 查看文章显示效果，酌情修改；
  3. 执行 hugo --minify --cleanDestinationDir；
  4. 进入 /public 目录，将新生成或编辑过的内容推送到 Github，此处不再赘述。

此时我们的博客就创建好了，但是创建文章内容的操作有一些复杂和繁琐，且本文中并未讲解关于 markdown 中插入图片的相关问题。所以在下文中，我们将简化操作步骤，并实现将图片也托管到 Github 的操作。
